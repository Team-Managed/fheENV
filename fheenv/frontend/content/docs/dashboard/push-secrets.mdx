# Push Secrets (Dashboard)

## How to Push

1. Navigate to your project → select an environment
2. Paste your `.env` content into the text area
3. Click **Push Secrets**

## What Happens

The entire encryption process runs **in your browser**:

1. **AES key generated** — `crypto.subtle.generateKey()` creates a random 256-bit key
2. **Content encrypted** — AES-256-GCM encrypts your entire `.env` text
3. **Blob uploaded** — encrypted ciphertext sent to IPFS via the server-side proxy
4. **CoFHE connected** — `@cofhe/sdk/web` connects to the Threshold Network
5. **Key FHE-encrypted** — the AES key is split into two 128-bit halves, each encrypted as `euint128`
6. **On-chain write** — `updateEnvironment()` stores the handles + CID on Sepolia

Your plaintext secrets **never leave your browser**. The server only sees encrypted blobs.

## Viewing Secrets

Click **View Secrets** to decrypt:

1. CoFHE creates a self-permit for your wallet
2. Threshold Network decrypts both `euint128` handles
3. AES key is reassembled in your browser
4. Encrypted blob fetched from IPFS and decrypted locally

## Editing Secrets

After viewing, click **Edit** to:

- Modify existing values
- Add new key-value pairs
- Delete keys

Click **Save** to re-encrypt with a fresh AES key and push the updated version.

## Version Control

Each push increments the environment version. The contract uses optimistic locking — if someone else pushes while you're editing, your push will fail with a version mismatch error. Refresh and try again.
