# CLI Commands

## `fheenv login`

Save a wallet private key in the local keyfile. The keyfile is plain JSON with permissions `0600`,
so protect the device and filesystem account that stores it.

```bash
fheenv login
fheenv login --key 0xABC123...
echo "0x..." | fheenv login
```

| Flag                     | Description                                                        |
| ------------------------ | ------------------------------------------------------------------ |
| `-k, --key <privateKey>` | 0x-prefixed key. Avoid it in shell history; omit it to read stdin. |

`FHEENV_PRIVATE_KEY` overrides the keyfile and is useful for a tightly scoped CI secret.

---

## `fheenv init`

Create an on-chain project and write `.fheenv.json`.

```bash
fheenv init --name "my-app" --pinata-jwt eyJ...
```

| Flag                       | Default              | Description              |
| -------------------------- | -------------------- | ------------------------ |
| `-n, --name <name>`        | required             | Project name             |
| `-r, --registry <address>` | Sepolia registry     | Contract address         |
| `--rpc <url>`              | PublicNode Sepolia   | RPC endpoint             |
| `--chain-id <id>`          | `11155111`           | Chain ID                 |
| `--pinata-jwt <jwt>`       | `$FHEENV_PINATA_JWT` | Pinata upload credential |
| `-e, --env <name>`         | `production`         | Default environment      |

---

## `fheenv push`

Encrypt and upload a local environment file.

```bash
fheenv push
fheenv push --env staging --file .env.staging
```

| Flag                | Default      | Description      |
| ------------------- | ------------ | ---------------- |
| `-e, --env <name>`  | `production` | Environment name |
| `-f, --file <path>` | `.env`       | Source file      |

It generates a fresh AES-256-GCM key, encrypts the file locally, uploads the encrypted blob to IPFS,
and writes FHE-encrypted key handles and the CID to the registry.

---

## `fheenv pull`

Decrypt an environment and write it locally.

```bash
fheenv pull
fheenv pull --env staging --output .env.staging.local
```

| Flag                  | Default      | Description         |
| --------------------- | ------------ | ------------------- |
| `-e, --env <name>`    | `production` | Environment to pull |
| `-o, --output <path>` | `.env.local` | Destination file    |

## `fheenv run`

Decrypt secrets and inject them into a child process without writing a file.

```bash
fheenv run -- node server.js
fheenv run --env staging -- npm start
```

## `fheenv team add`

Grant an address access to an environment.

```bash
fheenv team add --member 0xTeammate --env production
```

| Flag                     | Default      | Description    |
| ------------------------ | ------------ | -------------- |
| `-m, --member <address>` | required     | Wallet address |
| `-e, --env <name>`       | `production` | Environment    |

## `fheenv team remove`

Mark a member inactive in the registry.

```bash
fheenv team remove --member 0xFormer --env production
fheenv rotate --env production --file .env
```

<Callout type="warn">
  The contract cannot revoke existing FHE permissions. Run `fheenv rotate` immediately after removal
  to issue fresh ciphertext handles that exclude the removed wallet. Until rotation succeeds, that
  wallet can decrypt the old handles.
</Callout>

## `fheenv rotate`

Re-encrypt an environment with a new AES key and re-grant its current active members.

```bash
fheenv rotate
fheenv rotate --env staging --file .env.staging
```

| Flag                | Default      | Description           |
| ------------------- | ------------ | --------------------- |
| `-e, --env <name>`  | `production` | Environment to rotate |
| `-f, --file <path>` | `.env`       | Local source file     |

## `fheenv update`

Download and install the latest fheENV release.

```bash
fheenv update
```

## Environment Variables

| Variable             | Description                                 |
| -------------------- | ------------------------------------------- |
| `FHEENV_PRIVATE_KEY` | Raw wallet key; overrides the local keyfile |
| `FHEENV_PINATA_JWT`  | Pinata JWT used by `init`                   |
| `FHEENV_RPC`         | RPC URL override                            |
