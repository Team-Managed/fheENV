# Security Model

<Callout type="warn">
  Found a vulnerability? Do not open a public issue. See our [Security
  Policy](https://github.com/Team-Managed/fheENV/blob/main/SECURITY.md) for responsible disclosure.
</Callout>

## Core model

Environment content is encrypted locally with a fresh AES-256-GCM key. The encrypted blob is
uploaded to IPFS; the AES key is split into two FHE ciphertext handles stored on Sepolia. A
permitted wallet decrypts the handles through the threshold network and decrypts the blob locally.

## What is protected

| Threat                    | Current protection                              |
| ------------------------- | ----------------------------------------------- |
| IPFS or gateway access    | The blob is AES-256-GCM encrypted               |
| Public chain inspection   | The AES key is stored as FHE ciphertext handles |
| fheENV service compromise | No fheENV server stores plaintext               |
| Network interception      | HTTPS plus encrypted application data           |

## Important limits

- A compromised client can expose its key and any plaintext it decrypts.
- `~/.fheenv/wallet.json` is a plaintext private-key JSON file protected by filesystem mode `0600`;
  it is not passphrase-encrypted.
- The contract has no `revokeAllow` operation. `team remove` only marks a member inactive. You must
  run `fheenv rotate` with the source environment file to replace old FHE handles.
- Rotation cannot make plaintext or old encrypted blobs already copied by a former member disappear.
- The current contract supports project creation, owner addition, environment updates, grants, and
  revocations. It does not provide expiring grants, a Rotator role, scheduled rotation, or Lit/OIDC
  signing.

## Access control

```
Project owner ──► create projects, update environments, grant/revoke members, add owners
Member        ──► decrypt an environment when permitted on its current FHE handles
```

Owners are stored on-chain. Members are tracked per project and environment. The FHE permission on a
handle is the cryptographic gate, so rotating after a removal is essential.

## Data at rest

| Data                       | Location                | Protection                |
| -------------------------- | ----------------------- | ------------------------- |
| `.env` plaintext           | Local machine           | Local filesystem controls |
| Encrypted environment blob | IPFS                    | AES-256-GCM               |
| AES key handles            | Sepolia                 | FHE ciphertexts           |
| Wallet private key         | `~/.fheenv/wallet.json` | Filesystem mode `0600`    |

## Audit evidence

The dashboard exports public on-chain events: project creation, owner addition, environment updates,
access grants, and access revocations. It derives the transaction sender from the public chain
indexer. The contract does not emit a prior CID or IPFS unpin status, so those are not represented
as authoritative audit evidence.
