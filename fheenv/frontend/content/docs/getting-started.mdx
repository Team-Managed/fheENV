# Getting Started

## Prerequisites

- An Ethereum wallet with some Sepolia ETH (for gas)
- A [Pinata](https://app.pinata.cloud) account (free tier works) for IPFS storage

## Install the CLI

<Tabs items={["macOS / Linux", "Windows"]}>
  <Tab value="macOS / Linux">
    ```bash curl -fsSL https://raw.githubusercontent.com/Team-Managed/fheENV/main/install.sh | bash
    ```
  </Tab>
  <Tab value="Windows">
    ```powershell irm https://raw.githubusercontent.com/Team-Managed/fheENV/main/install.ps1 | iex
    ```
  </Tab>
</Tabs>

Verify:

```bash
fheenv --version
```

## First Push in 4 Steps

<Steps>
<Step>
### Authenticate

```bash
fheenv login
# Enter private key: 0x...
# Enter a passphrase to encrypt the local keyfile
```

<Callout type="warn">
Never pass your key as a CLI flag in production. Use the interactive prompt for developer machines,
or a tightly scoped platform secret for automation.
</Callout>
</Step>

<Step>
### Initialize a project

```bash
cd my-app
fheenv init --name "my-app" --pinata-jwt YOUR_PINATA_JWT
```

This creates `.fheenv.json` in your directory. Commit this file — it contains no secrets.

</Step>

<Step>
### Push your secrets

```bash title=".env"
DATABASE_URL=postgres://user:pass@host:5432/db
STRIPE_SECRET_KEY=sk_live_...
OPENAI_API_KEY=sk-proj-...
```

```bash
fheenv push --env production
```

Your `.env` is AES-encrypted locally, uploaded to IPFS, and the key is FHE-encrypted on-chain.

</Step>

<Step>
### Pull or run

<Tabs items={['Pull to file', 'Inject into process']}>
<Tab value="Pull to file">
```bash
fheenv pull --env production
# Writes to .env.local (chmod 600)
```
</Tab>
<Tab value="Inject into process">
```bash
fheenv run --env production -- node server.js
```
<Callout type="info">
Secrets exist only in the child process memory. Nothing touches the filesystem. This is the recommended approach.
</Callout>
</Tab>
</Tabs>
</Step>
</Steps>
